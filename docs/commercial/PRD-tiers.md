# 产品分层：体验版 vs 付费完整版（草案）

本文档固化「官网 / 体验版 / 付费完整版」在**功能与数据**上的边界，供研发与商务对齐。数值可在上线前按成本与风控再调一版。

## 角色与入口（概念）

| 角色 | 入口 | 说明 |
|------|------|------|
| 访客 | 官网 | 仅营销内容、小技巧、合规链接；可点「体验」进入练习子站 |
| 体验用户 | 练习站 · 体验模式 | 使用共享账号或匿名试用策略（见下）；能力受限 |
| 付费用户 | 练习站 · 已订阅 | 手机号登录（或手机号已绑定）；解锁完整能力 |
| 教师 | `?portal=teacher` | 不挂在官网；见 [teacher-portal-privacy.md](./teacher-portal-privacy.md) |

## 体验版（Demo / Trial）

**目标**：让访客在**不付费**的前提下感受流程（读题、录音、基础反馈之一），同时控制成本与滥用。

建议默认策略（与当前实现兼容的方向）：

| 维度 | 建议边界 | 说明 |
|------|----------|------|
| 登录方式 | 固定 `studentaccount` + 公开演示密码 **或** 官网一键「试用 token」（后续实现） | 共享账号会扩散，必须配频控与演示数据隔离 |
| Interview | 仅开放「体验包」题目：例如固定 **1 套（≤4 题）** 或教师预置的 demo 作业 | 不开放全量 `practice-bank` 浏览 |
| 录音提交 | 每日每 IP / 每设备 **≤ N 次**（如 5～10 次可配置） | 防止刷 ASR |
| 批改 / ASR | **二选一**：仅返回「已收到录音」+ 文本占位；或 **每日 1～2 次完整 ASR** | 与腾讯云/Ollama 成本挂钩 |
| Listen & Repeat | 开放 **1 组**或 **前 3 句** | 与现有多组布置逻辑可后续加「体验组号」白名单 |
| 数据归属 | 写入 `tenant_id=demo` 或单独 `demo_*` 表 | 与付费用户生产数据隔离，便于定期清理 |
| 教师布置 | 体验版 **不提供**「为真实班级布置」能力；或仅演示环境教师账号 | 避免 demo 污染正式作业 |

## 付费完整版（Paid）

**目标**：个人或机构付费后，用**手机号**建立可追责账号，解锁完整学习与批改能力。

| 维度 | 建议边界 | 说明 |
|------|----------|------|
| 身份 | 手机号 + 短信 OTP 登录；首次登录创建 `users` 扩展记录 | 与共享账号解耦 |
| Interview | 全量题库访问策略由你方定义：按订阅档位开放「话题包」或「全站」 | 若仍走「教师布置」，则学生端仍是姓名匹配作业 + 可选自主练习包 |
| 录音与批改 | 按订阅包含 **月度批改次数** 或 **无限**（高价档） | 需在服务端计次 |
| Listen & Repeat | 按档位开放组数 / 智聆调用次数 | 与 Python 后端配额一致 |
| 订单 | 微信/支付宝支付成功 → 服务端回调 → 写入 `orders` + `entitlements` | 以服务端为准，不信任前端 |

## 与当前系统的关系（摘要）

- 今日：**统一学生账号** + **姓名**匹配作业，已能避免「看到全部题库」——除非前端另有全量题库入口。
- 明日：在服务端增加 **`tier` / `entitlement`**（见 [data-model-draft.md](./data-model-draft.md)），所有昂贵接口（`submit-audio`、ASR、Ollama）先校验权益再执行。

## 待产品拍板的 3 个问题

1. 体验版 Interview 是「固定 4 题体验包」还是「跟当前老师布置走但限次」？
2. 付费是 **C 端按人** 还是 **B 端按校/按班**（影响发票与合同）？
3. 批改次数不足时：硬阻断还是降级为「仅保存录音不评分」？

定稿后，本文件可作为 v1 PRD 附件；版本号建议写在文件头 `v0.1`。
