# 中国大陆：主体、支付、短信 — 对接清单（可实施性）

本文档供与财务/法务/运营对齐，**不构成法律意见**；费率与资质以各平台最新审核为准。

## 1. 主体与前置条件

| 事项 | 说明 |
|------|------|
| 企业主体 | 微信/支付宝 **商户** 通常要求 **企业**（个体工商户视类目可能可行） |
| 对公账户 | 结算、提现、合同常见要求 |
| ICP 备案 | 若面向大陆用户提供 **经营性网站/应用**，按实际场景咨询是否需要备案及类目 |
| 隐私政策 / 用户协议 | 收集手机号、录音、作业数据、调用第三方 ASR 等需在协议中披露 |
| 类目与资质 | 若涉及「教育培训/在线课程」等，按支付与小程序/开放平台类目准备材料 |

## 2. 微信支付（示例路径）

- **产品**：Native / JSAPI / H5 等，按你的终端选型（浏览器 H5 常见 H5/公众号场景）。
- **技术**：统一下单 → 用户支付 → **异步通知**（callback）→ 服务端验签 → 更新订单与权益。
- **对接要点**：
  - `api_key`、商户号、证书等 **仅服务端**；
  - 回调 URL 公网可访问、HTTPS；
  - 幂等：同一 `out_trade_no` 重复通知只处理一次。

## 3. 支付宝（示例路径）

- **产品**：手机网站支付 / 当面付等。
- **技术**：类似：创建订单 → 支付 → **异步通知** → 验签 → 开通权益。
- **对接要点**：同样强调服务端验签与幂等。

## 4. 短信验证码（SMS OTP）

| 服务商类型 | 说明 |
|------------|------|
| 云厂商短信 | 阿里云、腾讯云等：模板 + 签名审核，按条计费 |
| 内容 | 短信模板需说明「验证码」用途，勿含营销违规话术 |

**技术要点**：

- 验证码 **服务端生成与校验**，有效期短（如 5–10 分钟）；
- 限频：同一手机号 **60s 内不能重复发**、每日上限；
- 防刷：图形验证码或 IP / 设备指纹（按投入选型）。

## 5. 预算与费率（量级）

- 短信：约 **0.03～0.05 元/条**（量大可谈，以合同为准）。
- 支付：按微信/支付宝 **行业费率**（例如千分之几级别，具体以签约为准）。
- 建议财务做 **单用户获客成本**：短信条数 + 支付手续费 + ASR/LLM 成本 + 服务器。

## 6. 与研发的对接顺序

1. 确定主体与商户号能开通的 **支付方式**（H5 / 小程序 / 仅扫码）。
2. 确定短信 **签名与模板** 审核通过后再联调。
3. 测试环境：使用平台提供的 **沙箱/沙盒**（若可用）；
4. 生产：小流量灰度 → 回调监控与对账。
