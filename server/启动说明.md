# Node 后端如何启动

## 正确命令（任选一种）

**方式 A：分两行输入**（推荐，避免粘贴出错）

```bash
cd "/Users/simonezhang/cursor/interview练习/server"
npm start
```

**方式 B：一行**（注意 `&&` 和引号后要有空格）

```bash
cd "/Users/simonezhang/cursor/interview练习/server" && npm start
```

### 常见错误

| 错误写法 | 问题 |
|---------|------|
| `cd "...server"npm start` | `server"` 和 `npm` 粘在一起，目录名错了，**不会**执行 `npm start` |
| 多行命令粘成一行 | 例如 `cd servernpm start` 中间缺空格 |

看到 **「已连接 Supabase」「后端运行在 http://localhost:3003」** 就表示启动成功。

## 端口被占用（EADDRINUSE）

默认端口见 `server/.env` 里的 `PORT`（一般为 **3003**）。

```bash
lsof -i :3003
kill <上面显示的 PID>
```

然后再 `npm start`。

## 与前端一致

`standalone.html` 里 `API_NODE` 需与这里端口一致（默认 `http://localhost:3003`）。

## 学生端登录失败

学生统一账号、密码由 **`server/.env`** 中的 `STUDENT_PORTAL_USERNAME`、`STUDENT_PORTAL_PASSWORD` 决定（未配置时默认为 `studentaccount` / `45678`），须与 **`standalone.html` 顶部的同名常量**一致。

1. 确认本机已启动 Node：`http://localhost:3003/api/health` 能返回 JSON。
2. 密码请输入 **`45678`**（若改过 `.env`，请用新密码）。
3. 若仍提示「密码错误」，在 `.env` 中显式写上 `STUDENT_PORTAL_PASSWORD=45678` 并重启 `npm start`；若改了密码，请同步修改 `standalone.html` 里的 `STUDENT_PORTAL_PASSWORD`。
4. 临时可用页面上的 **「离线登录」** 进入学生端（不连后端，无法拉取云端作业）。

## 学生 Interview 作业提交音频显示「保存失败」

Node 会把录音 **Base64** 写入 Supabase 表 `student_submissions` 的 **`audio_b64`**、**`ref_text`** 列。若未执行迁移，写入会失败。

1. 打开 **Supabase 控制台 → SQL Editor**，执行项目里的 **`server/supabase-assignments-audio.sql`**（为 `student_submissions` 增加 `audio_b64`、`ref_text`）。
2. 重启 Node：`cd server && npm start`。
3. 再让学生提交；若仍失败，看页面上 **完整错误文案**（服务端已返回数据库具体原因），并对照 Node 终端日志。

**常见原因**：未执行上述 SQL；录音文件过大（可缩短答题时间）；学生姓名与教师布置时不一致导致无权提交（会提示「无权提交」而非保存失败）。
