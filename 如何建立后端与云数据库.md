# 如何建立后端 + 云数据库（Supabase）

已改用 **Supabase** 作为云数据库（PostgreSQL），官网 https://supabase.com 在国内一般可访问。后端在 `server/` 目录，按下面步骤即可跑通。

---

## 一、申请 Supabase 云数据库（免费）

1. **打开官网**
   - 浏览器访问：https://supabase.com  
   - 点击 **Start your project**，用 GitHub 或邮箱注册并登录。

2. **新建项目**
   - 登录后点 **New Project**
   - **Organization** 选默认或新建一个
   - **Name** 随便填（如 `speaking-practice`）
   - **Database Password** 设一个数据库密码（自己记住，后面可改）
   - **Region** 选离你近的（如 Singapore）
   - 点 **Create new project**，等一两分钟创建完成。

3. **拿到连接信息**
   - 左侧点 **Settings**（齿轮）→ **API**
   - 记下两样：
     - **Project URL**：形如 `https://xxxxxxxx.supabase.co` → 填到 `.env` 的 `SUPABASE_URL`
     - **Project API keys** 里 **service_role** 的密钥（不要用 anon）→ 填到 `.env` 的 `SUPABASE_SERVICE_KEY`

4. **建表**
   - 左侧点 **SQL Editor** → **New query**
   - 打开项目里的 `server/supabase-tables.sql`，把里面整段 SQL 复制粘贴到编辑器
   - 点 **Run**，执行成功即可（会创建 `users` 和 `practice_records` 两张表）。
   - **教师/学生任务**：再执行 `server/supabase-assignments.sql`（为 `users` 增加 `role`，并创建 `assignments`、`assignment_targets`、`student_submissions`）。
   - **学生录音 + 教师批改**：在 assignments 迁移之后，再执行 `server/supabase-assignments-audio.sql`（为 `student_submissions` 增加 `audio_b64`、`ref_text`；否则学生提交录音、教师批改会报错）。

## 二、本地运行后端（需要先装 Node.js）

1. **安装 Node.js**  
   若未安装：https://nodejs.org/ 下载 LTS 并安装。

2. **配置环境变量**
   ```bash
   cd "/Users/simonezhang/cursor/interview练习/server"
   cp .env.example .env
   ```
   用记事本打开 `.env`，填入 Supabase 信息，例如：
   ```env
   SUPABASE_URL=https://xxxxxxxx.supabase.co
   SUPABASE_SERVICE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   PORT=3003
   CORS_ORIGIN=http://localhost:8080
   TEACHER_REGISTER_SECRET=自行设一长串随机口令
   PYTHON_BASE_URL=http://localhost:5001
   ```
   `CORS_ORIGIN` 填你打开前端的地址（如用 Python 开的是 `http://localhost:8080`）。  
   `TEACHER_REGISTER_SECRET`：教师账号**首次注册**时，登录页会要求填写与此一致的口令。  
   `STUDENT_PORTAL_USERNAME` / `STUDENT_PORTAL_PASSWORD`：学生端**统一登录账号**（默认 `studentaccount` / `45678`）；修改密码后需与前端 `standalone.html` 内常量保持一致。学生登录后在页面填写**姓名**查找作业，姓名须与教师布置任务时填写的一致。  
   `PYTHON_BASE_URL`：Python 口语服务地址；教师账号 **`zhangyufei`** 在任务详情里点「批改该题」时，Node 会把学生录音转发到 `PYTHON_BASE_URL/api/asr-eval` 并把结果写回数据库。

3. **安装依赖并启动**
   ```bash
   npm install
   npm start
   ```
   终端出现「已连接 Supabase 云数据库」「后端运行在 http://localhost:3003」即表示成功。

4. **前端与双端口**
   - `standalone.html` 内：**Node** `API_NODE`（与 `.env` 的 `PORT` 一致，当前示例 `http://localhost:3003`）负责登录、练习记录、教师布置/学生任务；**Python** `API_PYTHON`（默认 `http://localhost:5001`）负责题库 `GET /api/practice-bank` 与评测 `POST /api/asr-eval`。
   - 本地需**同时**运行：`cd server && npm start` 与 `cd oral-python-backend && python app.py`（或你的启动方式）。
   - **Interview 题库加速**：首次从 docx 解析后会生成项目根目录 `practice-bank.cache.json`；之后 Python 重启也会**直接读该文件**（docx 未改时），不再每次解析。前端 `standalone.html` 会**优先请求同目录的 `practice-bank.cache.json`**，无则再调 `GET /api/practice-bank`。可手动生成缓存：`cd oral-python-backend && python export_practice_bank_cache.py`（需已安装 `python-docx` 等依赖）。
   - **学生端**：`student.html` 或 `standalone.html?portal=student`（统一账号登录 → 输入姓名查作业）  
   - **教师端**：`teacher.html` 或 `standalone.html?portal=teacher`  
   - 学生与教师须从对应入口登录（账号 `role` 与入口不一致会提示换链接）。

## 三、后端接口说明（和云数据库的关系）

| 接口 | 作用 | 数据库 |
|------|------|--------|
| `POST /api/auth/login`（body 含 `expectedRole` / 教师首次注册 `teacherCode`） | 登录 / 注册 | 表 `users`（含 `role`） |
| `POST /api/practice` | 提交一次练习（选择题得分 + 口语文本） | 表 `practice_records` |
| `GET /api/practice?username=xxx` | 查询某用户的练习记录 | `practice_records` |
| `GET/POST /api/teacher/assignments` 等 | 教师查看/创建任务 | `assignments`、`assignment_targets` |
| `GET /api/student/assignments?studentName=姓名`（或 `username`） | 学生任务列表（姓名为教师布置时填写） | `assignment_targets`、`student_submissions` |
| `POST /api/student/submit` | 学生提交评测快照（body 含 `studentName` 或 `username`） | `student_submissions` |
| `POST /api/student/submit-audio` | 学生仅提交录音（base64），不直接调 Python | `student_submissions.audio_b64`、`ref_text` |
| `POST /api/teacher/evaluate-submission` | 仅用户 **`zhangyufei`** 可调用：拉取录音并转发 Python 评测，写入 `eval_data` | `student_submissions` |

云数据库 = Supabase 里你这个项目下的 PostgreSQL，所有用户和练习记录都保存在那里。

## 四、上线部署

- **云数据库**：继续用当前 Supabase 项目即可，无需再买数据库。
- **后端**：把 `server/` 部署到能跑 Node 的环境（云服务器、Railway、Render 等），在环境变量里填同样的 `SUPABASE_URL`、`SUPABASE_SERVICE_KEY`、`CORS_ORIGIN`（改为正式前端域名）。
- **前端**：把 `standalone.html` 放到任意静态托管，把里面的 **`API_NODE`**（及 `API_PYTHON`）改成你部署好的后端 / Python 地址；`API_NODE` 端口须与 Node 的 `PORT` 一致。

---

## 五、小结

| 步骤 | 说明 |
|------|------|
| 云数据库 | Supabase 免费建项目，在 SQL Editor 执行 `supabase-tables.sql` 建表 |
| 后端 | 在 `server/.env` 填 `SUPABASE_URL`、`SUPABASE_SERVICE_KEY`，执行 `npm install`、`npm start` |
| 前端 | `standalone.html` 里 `API_NODE` 指向 Node 后端、`API_PYTHON` 指向 Python，登录和提交会保存到 Supabase |

若 Supabase 官网在你网络下仍打不开，可考虑国内云数据库（如腾讯云、阿里云 PostgreSQL/MySQL），再在后端用对应 Node 驱动连接，接口保持不变即可。
